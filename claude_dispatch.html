Multi-Agent

Dispatch

복잡한 작업을 여러 에이전트에게 자동으로 분배하는 오케스트레이션. 병렬 처리로 더 빠르고 정확하게 목표를 달성합니다.

사용 방법

1

개념 이해 — Orchestrator & Sub-agents

Dispatch는 하나의 Orchestrator(조율자) Claude가 큰 목표를 받아 이를 작은 서브태스크로 분해하고, 각각을 독립적인 Sub-agent에게 병렬로 할당하는 패턴입니다. 각 Sub-agent는 자신의 태스크를 완료한 뒤 결과를 Orchestrator에게 반환합니다.

2

Agent Teams 활성화 (실험적 기능)

Claude Code에서 Agent Teams 기능을 활성화합니다. 현재 실험적 기능으로, Claude Code 설정에서 활성화할 수 있습니다.

# Claude Code 설정에서 활성화
# Settings > Experimental > Agent Teams
3

Sub-agent 생성 및 지시

Claude Code에서 자연어로 병렬 작업을 지시하거나, dispatch Skill을 사용해 태스크를 명시적으로 분배할 수 있습니다. Orchestrator는 각 Sub-agent의 진행 상황을 모니터링하고 결과를 통합합니다.

# 예시: 코드베이스 분석을 병렬 처리
"src/, tests/, docs/ 폴더를 동시에 분석하고
각 폴더의 품질 이슈를 요약해줘"
4

결과 취합 및 검토

모든 Sub-agent의 작업이 완료되면 Orchestrator가 결과를 취합해 최종 응답을 생성합니다. 각 단계의 진행 상황은 Claude Code 터미널에서 실시간으로 확인할 수 있습니다.

5

Dispatch Skill 활용

커뮤니티에서 제공하는 dispatch Skill을 설치하면 더 체계적인 병렬 에이전트 실행이 가능합니다. GitHub에서 bassimeledath/dispatch를 참고하세요.
