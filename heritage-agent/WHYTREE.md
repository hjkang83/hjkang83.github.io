# Heritage Agent Why Tree
> 유적지 여행자가 진짜 원하는 것이 무엇인지, "왜?"를 반복해서 파고든 결과

---

## Why Tree란?

사용자의 니즈에 대해 "왜 그걸 원하는가?"를 반복적으로 물어서 근본 욕구를 찾고, 각 욕구를 충족시킬 해결 수단을 도출하는 분석 방법이다.

---

## 분석 결과: 3가지 핵심 니즈

유적지 여행자의 니즈를 파고들면 크게 3가지 줄기로 나뉜다.

```
유적지 여행자가 원하는 것
│
├── 1. 정보 획득
├── 2. 함께 추억 만들기 (가이드 서비스)
└── 3. 헤매지 않기
```

---

## 줄기 1: 정보 획득

```
정보 획득
│
├─ WHY? 모르고 오면 아까움
│  │
│  └─ WHY? 매일 올 수 있는 게 아님
│          (유적지는 일상 공간이 아니라 특별한 방문이다.
│           한 번 올 때 최대한 알고 봐야 후회가 없다.)
│
└─ HOW? 정보 획득 방법
   │
   ├─ AI / 오디오 / 검색
   ├─ 유료 가이드 / 현지인
   └─ 현장 안내판 / 지인 / SNS
```

### 인사이트

여행자는 "정보 그 자체"를 원하는 게 아니다. **"다시 오기 어려운 곳에서 모르고 지나치는 게 아깝다"**는 감정이 근본 동기다. 이 아까움을 해소하려면 현장에서 바로 깊은 정보에 접근할 수 있어야 한다.

### Heritage Agent의 대응

| 기존 방법 | 문제점 | 우리의 해결 |
|-----------|--------|------------|
| AI/오디오/검색 | 일반적인 정보, 깊이 없음 | 발굴 보고서 기반 RAG로 전문 수준 정보 제공 |
| 유료 가이드/현지인 | 비용 부담, 일정 맞추기 어려움 | 무료 AI 가이드가 24시간 대기 |
| 현장 안내판/지인/SNS | 정보가 단편적, 개인화 안 됨 | 페르소나별 맞춤 설명 (어린이/일반/전문가) |

---

## 줄기 2: 함께 추억 만들기 (가이드 서비스)

```
가이드 서비스
│
├─ 함께 추억 만들기
│  │
│  └─ WHY? 나중에 돌이키며 행복
│          (여행의 가치는 현장에서 끝나지 않는다.
│           돌아와서 "그때 거기서 이랬지" 하며
│           꺼내보는 것이 진짜 행복이다.)
│
├─ 여행: 함께
│  │
│  └─ 일정 맞추기 스트레스
│     (동행자와 스케줄을 조율하는 것 자체가 피로)
│
└─ 여행: 혼자
   │
   └─ 나 혼자 가는 게 편함
      (일정 스트레스 없이 내 페이스대로 다닐 수 있다)
```

### 인사이트

여행자는 두 가지 상반된 욕구 사이에 있다. **함께여야 추억이 되지만, 혼자가 편하다.** 가이드 서비스는 이 딜레마를 풀어줄 수 있다. 혼자 가더라도 AI 가이드가 "함께하는 느낌"을 주고, 그 기록이 남으면 나중에 돌이킬 추억이 된다.

### Heritage Agent의 대응

| 딜레마 | 우리의 해결 |
|--------|------------|
| 혼자 가면 추억이 빈약하다 | AI 가이드가 대화 상대가 되어 "함께한 경험"을 만든다 |
| 함께 가면 일정 스트레스 | AI는 내 페이스에 맞춰주고, 일정 조율이 필요 없다 |
| 나중에 돌이킬 게 없다 | **지도 앨범이 자동으로 여행 기록을 남긴다** (사진 + AI 설명 + 위치) |

> **지도 앨범 기능의 근거가 바로 이 줄기에서 나온다.**
> "나중에 돌이키며 행복"하려면 기록이 남아야 한다.

---

## 줄기 3: 헤매지 않기

```
헤매지 않기
│
├─ WHY? 돈/시간 낭비가 아까움
│  │
│  └─ WHY? 시간은 한정적임
│          (여행 일정은 정해져 있다.
│           헤매는 시간 = 볼 수 있었던 것을 놓치는 시간)
│
└─ HOW? 헤매지 않는 방법
   │
   ├─ 엑셀 계획 / 운영시간 확인
   └─ 실시간 교통 / 동선 최적화
```

### 인사이트

여행자가 두려워하는 것은 길을 잃는 것 자체가 아니라, **"한정된 시간을 허비하는 것"**이다. 그리고 가장 뼈아픈 허비는 "바로 옆에 좋은 게 있었는데 몰라서 그냥 지나친 것"이다.

### Heritage Agent의 대응

| 기존 방법 | 문제점 | 우리의 해결 |
|-----------|--------|------------|
| 엑셀 계획/운영시간 | 사전 조사 시간 소요, 현장 변수 대응 불가 | GPS 기반 "근처 추천" 기능이 실시간으로 안내 |
| 실시간 교통/동선 최적화 | 네이버맵 등이 이미 잘함 (우리 영역 아님) | 교통이 아닌 **"콘텐츠 동선"**에 집중 — "여기서 500m에 가마터 유적이 있어요" |

## Why Tree → 기능 매핑 요약

```
WHY TREE 니즈              →  Heritage Agent 기능
─────────────────────────────────────────────────
모르고 지나치면 아까움      →  📸 사진 업로드 → AI 맞춤 설명
매일 올 수 없음            →  🗺️ 지도 앨범 (기록 영구 보존)
나중에 돌이키며 행복        →  🗺️ 지도 앨범 (사진+설명 자동 저장)
혼자가 편하지만 추억 필요   →  🤖 AI 가이드 (혼자여도 대화 경험)
시간 낭비가 아까움          →  📍 근처 추천 (놓칠 뻔한 장소 안내)
눈높이에 맞는 정보 필요     →  👶🧑🎓 페르소나별 설명
```

---

## 결론

Why Tree 분석을 통해 확인한 것: Heritage Agent의 모든 핵심 기능은 **여행자의 실제 감정과 욕구**에서 출발한다.

- **정보 획득** → 가이드 모드 (AI 설명)
- **추억 만들기** → 지도 앨범 (기록 자동 저장)
- **헤매지 않기** → 근처 추천 (GPS 기반 동선 안내)

기능을 추가하거나 뺄 때는 항상 이 Why Tree로 돌아와서, "이 기능이 세 줄기 중 어디에 해당하는가?"를 확인한다. 어디에도 해당하지 않으면 빼야 할 기능이다.
